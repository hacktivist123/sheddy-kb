# APIs and data models

This cluster captures the design thinking around API contracts, request/response shape, and storage models. It connects browser-side API consumption to backend persistence choices.

## Core ideas

- APIs as contracts between systems
- REST as a practical interface style
- Graph databases versus relational databases
- Type-safe database operations in Go
- The shape of data as an architectural decision
